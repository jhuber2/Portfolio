Multiplayer Battleship Game

Overview

This project is a multiplayer Battleship game API built with FastAPI that allows players to create games, join matches, place ships, and take turns in real time. It is interesting because it manages game rules, turn logic, and shared game state across multiple players through a clean REST API. I designed the backend around structured endpoints and persistent game state so the system could reliably enforce gameplay rules and support repeatable testing.

Key Features

  • Multiplayer game creation and player joining through API endpoints
  • Turn-based gameplay enforcement to preserve valid game flow
  • Ship placement validation to prevent invalid board states
  • Persistent game state for reliable match progression and testing
  • API-driven architecture designed around structured REST interactions

Tech Stack

  • Python
  • FastAPI
  • PostgreSQL
  • SQLAlchemy
  • Docker
